### 方案 A：Vercel 部署（推荐）

#### 前置准备
1. 一个 GitHub 账号（免费注册：https://github.com
2. 一个 Vercel 账号（用 GitHub 登录，免费）：https://vercel.com

#### 步骤 1：上传代码到 GitHub

1. 在 GitHub 创建一个新仓库（比如叫 `wuli-app`）
2. 把本地代码推送到仓库

```bash
cd 你的项目目录
git init
git add .
git commit -m "init: 物历应用"
git branch -M main
git remote add origin https://github.com/你的用户名/wuli-app.git
git push -u origin main
```

#### 步骤 2：在 Vercel 导入项目

1. 打开 https://vercel.com 登录
2. 点击 **"Add New..."** → **"Project"**
3. 选择你刚创建的 GitHub 仓库
4. 点击 **"Import"**

#### 步骤 3：配置部署参数

Vercel 会自动识别 Node.js 项目，保持默认即可：
- **Framework Preset**: `Other`
- **Build Command**: （留空）
- **Output Directory**: （留空）
- **Install Command**: （留空，自动 npm install）

#### 步骤 4：配置环境变量（AI API Key）

1. 在部署设置页面，找到 **"Environment Variables"**
2. 添加一个变量：
   - **Name**: `AI_API_KEY`
   - **Value**: 你的 API Key（比如 DeepSeek 的 sk-xxxx）
   - **Environment**: 勾选全部勾选（Production, Preview, Development）
3. 点击 **"Add"**

#### 步骤 5：点击 Deploy

点击 **"Deploy"** 按钮，等待 1-2 分钟，部署完成！

部署成功后，你会得到一个类似 `https://wuli-app-xxx.vercel.app 的域名，这就是你的公开体验链接。

#### 步骤 6：绑定自定义域名（可选）

如果有自己的域名：
1. 进入项目 → **Settings** → **Domains**
2. 输入你的域名，按提示配置 DNS

---

### 方案 B：Netlify 部署

1. 注册 Netlify：https://netlify.com
2. 点击 **"Add new site"** → **"Import an existing project"**
3. 选择 GitHub，授权并选择仓库
4. 配置：
   - Build command: 留空
   - Publish directory: `.`
5. 在 **Site settings** → **Environment variables** 添加 `AI_API_KEY`
6. 点击 Deploy

### 方案 C：国内云平台（阿里云/腾讯云）

如果评审主要在国内，建议使用国内平台：

#### 阿里云函数计算 FC
1. 开通函数计算服务

5. 配置环境变量：`AI_API_KEY`

### 方案 A：PWA（渐进式 Web 应用）⭐⭐⭐⭐⭐

**最简单，零开发成本，用户在浏览器里点"添加到桌面"就有 App 体验。**

已配置完成！项目已添加 PWA 支持：
- ✅ manifest.json（应用清单）
- ✅ 应用图标（SVG）
- ✅ 全屏显示模式（像原生App
- ✅ 主题色匹配

#### 使用方法：
1. 用手机 Chrome 浏览器打开部署好的网站
2. 点击浏览器菜单 → **"添加到主屏幕"** 或 **"安装应用"**
3. 桌面会出现"物历"图标，点击即可像App一样全屏打开

#### 优点：
- 零安装成本，扫码即用
- 自动更新，不需要发版
- 支持 iOS 和 Android 双端
- 体积为 0，不占存储空间

### 方案 C：Android Studio WebView 项目（最正规）⭐⭐⭐

如果你需要一个正规的 APK 安装包，可以用 Android Studio 构建。

#### 项目结构（最小化版本）

```
WuliApp/
├── app/
│   ├── build.gradle
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── java/com/wuli/app/MainActivity.java
│       └── res/
│           ├── layout/activity_main.xml
│           ├── mipmap/ （图标）
│           └── values/strings.xml
├── build.gradle
└── settings.gradle
```

#### 核心代码示例

**AndroidManifest.xml**
```xml
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <uses-permission android:name="android.permission.INTERNET" />
    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="物历"
        android:supportsRtl="true"
        android:theme="@style/Theme.AppCompat.Light.NoActionBar">
        <activity
            android:name=".MainActivity"
            android:exported="true"
            android:configChanges="orientation|screenSize|keyboardHidden">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>
```

**MainActivity.java**
```java
package com.wuli.app;

import android.os.Bundle;
import android.webkit.WebSettings;
import android.webkit.WebView;
import android.webkit.WebViewClient;
import androidx.appcompat.app.AppCompatActivity;

public class MainActivity extends AppCompatActivity {
    private WebView webView;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        webView = findViewById(R.id.webview);
        WebSettings settings = webView.getSettings();
        settings.setJavaScriptEnabled(true);
        settings.setDomStorageEnabled(true);
        settings.setLoadWithOverviewMode(true);
        settings.setUseWideViewPort(true);
        settings.setBuiltInZoomControls(false);
        settings.setCacheMode(WebSettings.LOAD_DEFAULT);

        webView.setWebViewClient(new WebViewClient() {
            @Override
            public boolean shouldOverrideUrlLoading(WebView view, String url) {
                view.loadUrl(url);
                return true;
            }
        });

        webView.loadUrl("https://你的部署域名");
    }

    @Override
    public void onBackPressed() {
        if (webView.canGoBack()) {
            webView.goBack();
        } else {
            super.onBackPressed();
        }
    }
}
```

**activity_main.xml**
```xml
<?xml version="1.0" encoding="utf-8"?>
<WebView xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/webview"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />
```

#### 构建步骤：
1. 下载 Android Studio：https://developer.android.com/studio
2. 创建新项目 → Empty Activity
3. 替换上面的代码
4. 把 `webView.loadUrl()` 改成你部署的域名
5. Build → Build APK(s)
6. 生成的 APK 可以安装到手机

## 三、AI API 配置

### 支持的模型服务商

| 服务商 | 模型名称 | 官网 | 备注 |
|--------|---------|------|------|
| **DeepSeek** | deepseek-chat | https://platform.deepseek.com | 推荐，性价比高 |
| 通义千问 | qwen-turbo | https://dashscope.aliyun.com | 阿里云 |
| 智谱清言 | glm-3-turbo | https://open.bigmodel.cn | 智谱AI |
| 豆包 | doubao-pro-4k | https://www.volcengine.com | 字节跳动 |

### 配置方法

#### 获取 API Key（以 DeepSeek 为例）：
1. 注册 DeepSeek 开放平台：https://platform.deepseek.com
2. 进入 API Key 管理
3. 创建新的 API Key
4. 复制 sk- 开头的密钥

#### 配置到部署平台：
1. 进入 Vercel/Netlify 后台
2. 找到 Environment Variables
3. 变量名：`AI_API_KEY`
4. 值：粘贴你的 API Key
5. 保存并重新部署

### 代码中的配置（如需更换模型服务商）

打开 `server.js`，修改这几个变量：

```javascript
const AI_API_URL = 'https://api.deepseek.com/v1/chat/completions';
const AI_MODEL = 'deepseek-chat';
```

换成对应服务商的 API 地址和模型名即可（都是 OpenAI 兼容格式）。

## 四、成本控制说明

### 当前限流配置

| 限制项 | 默认值 | 说明 |
|--------|--------|------|
| 每日总调用上限 | 150 次 | 全局，控制整体成本 |
| 单IP每小时上限 | 8 次 | 防止单用户滥用 |
| 单次回复 max_tokens | 500 | 避免单条回复太长 |
| 用户输入长度限制 | 300 字 | 减少 token 消耗 |

### ⚠️ 重要提醒

**当前限流实现方式：内存计数器**

```
注意：在 Vercel/Netlify 这类 Serverless 环境下，
内存计数器会因为冷启动而被重置，
限流可能不精确，实际调用次数可能会超过设定值。
```

#### 如果需要严格控制成本，建议：

1. **把每日上限设小一点**（比如 80-100 次）
   - 修改 server.js 里的 `DAILY_LIMIT` 变量

2. **在 API 服务商后台设置消费上限**
   - DeepSeek 后台可以设置月度消费限额
   - 这样即使代码限流失效，也不会超预算

3. **后续升级方案：接 Redis/Upstash**
   - 用外部 KV 存储做计数
   - 限流会更精确
   - 但需要额外接入成本

### 修改限流配置

打开 `server.js`，找到这几行可以修改：

```javascript
const DAILY_LIMIT = 150;        // 每日总上限
const IP_HOURLY_LIMIT = 8;       // 每IP每小时上限
const MAX_TOKENS = 500;        // 单次回复 token 上限
```
