# 物历首次使用引导设计方案

**日期**: 2026-06-29  
**状态**: 设计中  
**版本**: 1.0

---

## 1. 概述与目标

### 1.1 功能目标
为物历应用设计首次使用引导系统，帮助新用户快速了解核心功能，提升用户留存和功能使用率。

### 1.2 设计原则
- **场景化**: 根据用户选择的视角（新婚/二手/家庭守护）显示不同的引导内容
- **渐进式**: 先欢迎提示，再逐步引导，避免信息过载
- **可发现**: 引导模式下高亮提示"可点击查看详情"，之后保持持久可点击
- **轻量化**: 使用底部弹出卡片，不遮挡主要内容，操作后可继续浏览

## 2. 用户流程

### 2.1 完整用户流程
```
用户打开应用
    ↓
注册/登录（首次使用）
    ↓
选择体验视角 ← 【选择视角弹窗】
    ↓
显示欢迎引导卡片 ← 【欢迎弹窗】
    ↓
用户点击"开始探索"
    ↓
进入4步渐进引导 ← 【遮罩+高亮+箭头】
    ↓
引导完成 → 标记已引导 → 进入首页
    ↓
用户日常使用 → 可随时点击各模块查看详情
```

### 2.2 触发条件
- `AppState.hasSeenOnboarding === false`
- 用户已完成视角选择
- 用户已登录

## 3. 视角专属引导内容

### 3.1 新婚视角（newlywed）
针对新婚夫妻共建小家的场景：
1. **物品档案** - "为你们的共同物品建立档案，说明书、购入日期都能记录"
2. **AI管家** - "电饭煲不工作了？先问问AI管家，自检能解决大部分问题"
3. **闲置流转** - "闲置的婚庆物品可以分享给邻居，还能增进邻里关系"
4. **家庭守护** - "设置双方父母的用药提醒，远在千里也能关心到位"

### 3.2 二手视角（secondhand）
针对闲置流转、社区交易的场景：
1. **闲置流转** - "拍张照就能发布闲置，买家直接联系你"
2. **物品档案** - "二手物品更要看重保养记录，这些信息都要保存好"
3. **AI管家** - "不知道该卖多少钱？AI帮你智能估价"
4. **家庭守护** - "家人健康也要关注，设置用药提醒更安心"

### 3.3 家庭守护视角（family）
针对远程照顾家人的场景：
1. **家庭守护** - "添加爸妈的常用物品和药品，实时掌握健康状态"
2. **物品档案** - "家人的贵重物品也能一起管理，有据可查"
3. **AI管家** - "设备故障不会处理？AI管家帮你远程诊断"
4. **闲置流转** - "闲置物品流转出去，腾出更多生活空间"

## 4. UI组件设计

### 4.1 欢迎引导卡片
**触发时机**: 视角选择完成后立即显示  
**位置**: 屏幕中央居中显示  
**尺寸**: 宽度90%，最大360px  
**内容**:
```
┌─────────────────────────────┐
│         🎉                 │
│    欢迎来到物历！           │
│                             │
│  你选择了 [视角名称] 视角    │
│  让我带你快速了解一下吧     │
│                             │
│  ┌───────────────────────┐ │
│  │    开始探索 →          │ │
│  └───────────────────────┘ │
│                             │
│  [跳过引导]                 │
└─────────────────────────────┘
```

### 4.2 渐进引导遮罩
**触发时机**: 点击"开始探索"后  
**组成元素**:
- 半透明黑色遮罩（rgba(0,0,0,0.6)）
- 目标模块挖空显示（带圆角边框高亮）
- 引导气泡（指向挖空区域）
- 底部步骤指示器
- 跳过按钮

**引导气泡内容**:
```
┌─────────────────────────────┐
│  ← 步骤2/4                  │
│                             │
│  [功能图标]                 │
│  [功能名称]                 │
│  [功能说明，1-2句话]        │
│                             │
│  ← 上一步    下一步 →       │
└─────────────────────────────┘
```

### 4.3 功能详情弹窗
**触发时机**: 任何时候点击功能模块  
**位置**: 底部弹出（类似现有modal）  
**尺寸**: 宽度100%，圆角顶部32px  
**内容结构**:
```
┌─────────────────────────────┐
│         ─────              │ ← 拖动把手
│                             │
│  [功能大图标/插画]          │
│  [功能名称]                 │
│                             │
│  [功能简介]                 │
│                             │
│  ┌─────────────────────┐   │
│  │ 💡 使用技巧          │   │
│  │ [技巧1]              │   │
│  │ [技巧2]              │   │
│  └─────────────────────┘   │
│                             │
│  ┌─────────────────────┐   │
│  │ 🎯 操作建议          │   │
│  │ [建议链接/按钮]      │   │
│  └─────────────────────┘   │
│                             │
│  [我知道了]                 │
└─────────────────────────────┘
```

### 4.4 步骤指示器
**位置**: 屏幕底部，距底部导航56px + 20px  
**样式**: 
- 圆点 × 4，当前步骤高亮
- 当前: 实心圆点，主题色
- 其他: 空心圆点，灰色
- 尺寸: 当前12px，其他8px

```javascript

  // ... 现有状态

  onboardingPerspective: null,    // 引导时的视角
  onboardingShownModules: [],     // 已展示详情的模块
}
```

### 6.2 引导数据配置
```javascript
const ONBOARDING_GUIDES = {
  newlywed: [
    {
```

### 6.3 关键函数
```javascript
function startOnboarding(perspective) {
  AppState.hasSeenOnboarding = true;
  AppState.onboardingPerspective = perspective;
  AppState.onboardingStep = 1;
  showWelcomeModal();
}

function showWelcomeModal() {
  // 显示欢迎卡片
}

function showGuideStep(step) {
  // 显示遮罩+高亮+引导气泡
}

function showModuleDetail(moduleName) {
  // 显示功能详情弹窗
}

function skipOnboarding() {
  // 跳过引导，直接进入首页
  AppState.onboardingStep = 0;
  renderPage();
}

function completeOnboarding() {
  // 完成引导
  AppState.onboardingStep = 0;
  localStorage.setItem('hasSeenOnboarding', 'true');
  renderPage();
}
```

## 7. 交互细节

### 7.1 引导流程
1. 用户点击"开始探索" → 隐藏欢迎卡片 → 显示遮罩和第1步
2. 点击"下一步"或点击高亮区域 → 进入下一步
3. 点击"上一步" → 返回上一步
4. 最后一步"下一步" → 完成引导
5. 点击"跳过引导" → 直接进入首页

### 7.2 点击详情
1. 引导期间：点击高亮模块 → 显示详情弹窗（保持引导状态）
2. 引导结束后：点击任意标记模块 → 显示详情弹窗
3. 详情弹窗点击外部或"我知道了" → 关闭弹窗

### 7.3 引导高亮目标
每一步引导聚焦一个主要模块：
- 步骤1: 物品档案卡片
- 步骤2: AI管家入口（首页右上角或提醒卡片）
- 步骤3: 闲置流转卡片
- 步骤4: 家庭守护按钮

## 9. 兼容性考虑

### 9.1 现有功能兼容
- 不影响现有的 `showModal()` / `hideModal()` 系统
- 引导遮罩使用独立的z-index层级
- 点击事件正确穿透/阻止

### 9.2 响应式适配
- 引导气泡在小屏幕上适当缩小
- 详情弹窗在安全区域内显示

### 9.3 状态持久化
- `hasSeenOnboarding` 存储到 localStorage
- 用户清除缓存后可能重新显示引导
